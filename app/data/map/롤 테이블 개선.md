# 세션 롤 테이블 개선안

# 개요

기존 의뢰 롤 테이블은 단일 표에서 의뢰 유형 하나를 직접 뽑는 방식에 가깝다. 이 방식은 빠르게 사용할 수 있지만, 의뢰의 성격이 복잡해질수록 관리가 어려워진다.

개선된 세션 롤 테이블은 모험가의 세 가지 길인 **개척자**, **해결사**, **사냥꾼**을 최상위 분류로 사용한다. 이후 대분류, 소분류, 현장 조건, 미지 요소, 충돌 요소, 실패 조건을 단계적으로 조합하여 하나의 세션 공고를 생성한다.

이 구조의 목적은 단순히 “무엇을 하러 가는가”를 정하는 것이 아니라, “왜 자유 탐사 연맹이 이 사안을 공고로 올렸는가”를 명확히 만드는 데 있다.

# 1. 최종 생성 구조

세션 공고는 다음 순서로 생성한다.

```text
1. 대표 경로 결정
2. 대분류 결정
3. 소분류 결정
4. 현장 배경 결정
5. 미지 요소 결정
6. 충돌 요소 결정
7. 회수·기록 대상 결정
8. 실패 조건 결정
9. 후속 재분류 가능성 결정
10. 공고 템플릿 출력
```

개발 시에는 각 단계를 독립 테이블로 분리한다.
이렇게 하면 새로운 의뢰 유형이 추가되어도 전체 구조를 다시 짜지 않아도 된다.

function generateQuest(input?: {
  route?: AdventurerRoute;
  categoryId?: string;
  difficulty?: QuestDifficulty;
  allowClassified?: boolean;
}): GeneratedQuest {
  const route = input?.route ?? rollWeighted(routes.filter(routeFilter(input)));

  const category = input?.categoryId
    ? findCategory(input.categoryId)
    : rollWeighted(categories.filter((item) => item.route === route));

  const subCategory = rollWeighted(
    subCategories.filter((item) => item.categoryId === category.id)
  );

  const site = rollWeighted(
    sites.filter((item) => item.allowedRoutes.includes(route))
  );

  const mystery = rollWeighted(
    mysteries.filter((item) => isCompatible(item, category, subCategory, site))
  );

  const conflict = rollWeighted(conflicts);
  const target = rollWeighted(targets.filter((item) => isTargetCompatible(item, subCategory)));
  const failCondition = rollWeighted(failConditions);

  const difficulty = input?.difficulty ?? calculateDifficulty({
    route,
    category,
    subCategory,
    site,
    mystery,
    conflict,
  });

  return buildQuest({
    route,
    category,
    subCategory,
    site,
    mystery,
    conflict,
    target,
    failCondition,
    difficulty,
  });
}

## 4.2 가중치 굴림 함수

```ts
export interface WeightedItem {
  weight: number;
}

export function rollWeighted<T extends WeightedItem>(items: T[]): T {
  const total = items.reduce((sum, item) => sum + item.weight, 0);
  let roll = Math.random() * total;

  for (const item of items) {
    roll -= item.weight;
    if (roll <= 0) return item;
  }

  return items[items.length - 1];
}
```

## 4.3 호환성 규칙

```ts
function isCompatible(
  mystery: MysteryTableItem,
  category: QuestCategory,
  subCategory: QuestSubCategory,
  site: SiteTableItem
): boolean {
  const categoryText = `${category.name} ${subCategory.name} ${site.tags.join(' ')}`;

  if (mystery.tags.includes('council')) {
    return category.route === 'CLASSIFIED' || categoryText.includes('룬') || categoryText.includes('기록');
  }

  if (mystery.tags.includes('angra')) {
    return category.route === 'PIONEER' || site.tags.includes('angra');
  }

  if (mystery.tags.includes('monster')) {
    return category.route === 'HUNTER' || categoryText.includes('생태') || categoryText.includes('사냥');
  }

  if (mystery.tags.includes('dungeon')) {
    return categoryText.includes('던전') || site.tags.includes('dungeon');
  }

  return true;
}
```

난이도는 단순 전투 난이도가 아니라, 공고의 위험도와 절차 복잡도를 함께 계산한다.

export interface DifficultyScoreInput {
  route: AdventurerRoute;
  site: SiteTableItem;
  mystery: MysteryTableItem;
  conflictId: string;
}

export function calculateDifficulty(input: DifficultyScoreInput): QuestDifficulty {
  let score = 0;

  if (input.route === 'PIONEER') score += 3;
  if (input.route === 'CLASSIFIED') score += 3;
  if (input.route === 'HUNTER') score += 2;
  if (input.route === 'SOLVER') score += 1;

  if (input.site.tags.includes('angra')) score += 3;
  if (input.site.tags.includes('dungeon')) score += 2;
  if (input.site.tags.includes('civilian')) score += 1;

  if (input.mystery.tags.includes('council')) score += 3;
  if (input.mystery.tags.includes('rune')) score += 2;
  if (input.mystery.tags.includes('core')) score += 2;
  if (input.mystery.tags.includes('monster')) score += 1;

  if (input.conflictId === 'conflict_time') score += 1;
  if (input.conflictId === 'conflict_infection') score += 2;
  if (input.conflictId === 'conflict_route_loss') score += 2;

  if (score <= 4) return 'LOW';
  if (score <= 7) return 'MIDDLE';
  if (score <= 10) return 'HIGH';
  return 'SPECIAL';
}

```ts
import type { WeightedItem } from '../types/quest';

export function rollWeighted<T extends WeightedItem>(items: T[]): T {
  if (items.length === 0) {
    throw new Error('rollWeighted: items must not be empty.');
  }

  const totalWeight = items.reduce((sum, item) => sum + item.weight, 0);

  if (totalWeight <= 0) {
    throw new Error('rollWeighted: total weight must be greater than 0.');
  }

  let roll = Math.random() * totalWeight;

  for (const item of items) {
    roll -= item.weight;

    if (roll <= 0) {
      return item;
    }
  }

  return items[items.length - 1];
}

의뢰 유형 1개를 바로 굴린다.

개선 방식:
모험가의 길을 먼저 정하고,
그 길에 맞는 대분류와 소분류를 선택한 뒤,
현장 조건과 미지 요소를 조합해 세션 공고를 만든다.
```

이 방식은 다음 장점을 가진다.

```text
1. 개척자, 해결사, 사냥꾼의 역할이 명확해진다.
2. 문명권 안전 관리는 해결사 공고로 안정적으로 분류된다.
3. 앙그라권 공고는 개척자 공고로 분리된다.
4. 몬스터 사냥은 사냥꾼 공고로 독립된다.
5. 기밀 공고는 일반 공고로 위장하거나 캠페인 진행도에 따라 해금할 수 있다.
6. 하나의 소분류에 여러 현장 변수와 미지 요소를 붙여 반복 사용성을 높일 수 있다.
7. 실제 웹앱이나 롤테이블 프로그램으로 바로 구현하기 쉽다.
```
