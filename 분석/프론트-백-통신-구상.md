# 프론트엔드-백엔드 통신 구상

> **FastAPI (백엔드) ↔ Nuxt 4 / Vue 3 (프론트엔드) 통신 아키텍처**

## 기술 스택 개요

### 백엔드 (FastAPI)
- **프레임워크**: FastAPI (Python)
- **통신**: REST API
- **데이터 형식**: JSON
- **인증**: JWT (추후 구현)

### 프론트엔드 (Nuxt 4 / Vue 3)
- **프레임워크**: Nuxt 4.2.2
- **UI 프레임워크**: Vue 3.5.27
- **상태 관리**: Pinia 3.0.4
- **HTTP 클라이언트**: Axios 1.13.2 + Nuxt의 useFetch/useAsyncData
- **폼 관리**: vee-validate + @vee-validate/zod (설치 필요)
- **스키마 검증**: Zod (설치 필요)
- **유틸리티**: VueUse 14.1.0

## 통신 라이브러리 구성

### 1. HTTP 클라이언트: Axios + Nuxt Composables

**선택 이유:**
- Nuxt의 `useFetch`/`useAsyncData`는 SSR 지원 및 자동 캐싱 제공
- Axios는 인터셉터, 에러 처리, 요청 취소 등 강력한 기능 제공
- 두 가지를 상황에 맞게 조합하여 사용

**사용 전략:**
```typescript
// 1. SSR이 필요한 페이지 데이터 로딩: useFetch
const { data, pending, error, refresh } = await useFetch('/api/traits', {
  baseURL: useRuntimeConfig().public.apiBase
})

// 2. 클라이언트 사이드 mutation (생성, 수정, 삭제): axios
const api = useAxios()
await api.post('/api/traits', traitData)

// 3. 복잡한 에러 처리가 필요한 경우: axios + interceptor
```

**Axios 설정 위치:**
- `app/utils/axios.ts` - Axios 인스턴스 생성 및 설정
- `app/composables/useAxios.ts` - Axios를 사용하는 composable

### 2. 상태 관리: Pinia

**선택 이유:**
- Vue 3 공식 상태 관리 라이브러리

## 상세 구현 계획

### 1. Axios 설정 (`app/utils/axios.ts`)

```typescript
import axios, { type AxiosInstance, type AxiosError } from 'axios'

export function createAxiosInstance(): AxiosInstance {
  const config = useRuntimeConfig()
  
  const instance = axios.create({
    baseURL: config.public.apiBase,
    timeout: 10000,
    headers: {
      'Content-Type': 'application/json',
    },
  })

  // Request Interceptor
  instance.interceptors.request.use(
    (config) => {
      // JWT 토큰 추가 (추후 구현)
      const token = useCookie('auth_token').value
      if (token) {
        config.headers.Authorization = `Bearer ${token}`
      }
      return config
    },
    (error) => {
      return Promise.reject(error)
    }
  )

  // Response Interceptor
  instance.interceptors.response.use(
    (response) => response,
    (error: AxiosError) => {
      // 에러 처리
      const statusCode = error.response?.status
      
      if (statusCode === 401) {
        // 인증 실패 - 로그인 페이지로 리다이렉트
        navigateTo('/login')
      } else if (statusCode === 403) {
        // 권한 없음
        console.error('권한이 없습니다.')
      } else if (statusCode === 404) {
        // 리소스 없음
        console.error('요청한 리소스를 찾을 수 없습니다.')
      } else if (statusCode && statusCode >= 500) {
        // 서버 에러
        console.error('서버 에러가 발생했습니다.')
      }
      
      return Promise.reject(error)
    }
  )

  return instance
}
```

### 2. Axios Composable (`app/composables/useAxios.ts`)

```typescript
import type { AxiosInstance } from 'axios'

let axiosInstance: AxiosInstance | null = null

export function useAxios(): AxiosInstance {
  if (!axiosInstance) {
    axiosInstance = createAxiosInstance()
  }
  return axiosInstance
}
```

### 4. Pinia Store 예시 (`app/stores/trait.ts`)

```typescript
import { defineStore } from 'pinia'
import { traitService } from '~/services/trait.service'
import type { Trait, TraitSearchParams } from '~/types'

export const useTraitStore = defineStore('trait', () => {
  // State
  const traits = ref<Trait[]>([])
  const currentTrait = ref<Trait | null>(null)
  const loading = ref(false)
  const error = ref<Error | null>(null)
  const pagination = ref({
    total: 0,
    pageNo: 1,
    pageSize: 20,
  })

  // Actions
  async function fetchTraits(params: TraitSearchParams) {
    loading.value = true
    error.value = null
    try {
      const response = await traitService.getTraits(params)
      traits.value = response.items
      pagination.value.total = response.total_cnt
    } catch (e) {
      error.value = e as Error
      throw e
    } finally {
      loading.value = false
    }
  }

  async function fetchTrait(traitNo: number) {
    loading.value = true
    error.value = null
    try {
      currentTrait.value = await traitService.getTrait(traitNo)
    } catch (e) {
      error.value = e as Error
      throw e
    } finally {
      loading.value = false
    }
  }

  async function createTrait(trait: Partial<Trait>) {
    loading.value = true
    error.value = null
    try {
      const newTrait = await traitService.createTrait(trait)
      traits.value.push(newTrait)
      return newTrait
    } catch (e) {
      error.value = e as Error
      throw e
    } finally {
      loading.value = false
    }
  }

  async function updateTrait(traitNo: number, trait: Partial<Trait>) {
    loading.value = true
    error.value = null
    try {
      const updatedTrait = await traitService.updateTrait(traitNo, trait)
      const index = traits.value.findIndex(t => t.trait_no === traitNo)
      if (index !== -1) {
        traits.value[index] = updatedTrait
      }
      return updatedTrait
    } catch (e) {
      error.value = e as Error
      throw e
    } finally {
      loading.value = false
    }
  }

  async function deleteTrait(traitNo: number) {
    loading.value = true
    error.value = null
    try {
      await traitService.deleteTrait(traitNo)
      traits.value = traits.value.filter(t => t.trait_no !== traitNo)
    } catch (e) {
      error.value = e as Error
      throw e
    } finally {
      loading.value = false
    }
  }

  return {
    // State
    traits,
    currentTrait,
    loading,
    error,
    pagination,
    // Actions
    fetchTraits,
    fetchTrait,
    createTrait,
    updateTrait,
    deleteTrait,
  }
})
```

### 5. Nuxt 페이지에서의 사용 (`app/pages/traits/index.vue`)

```vue
<script setup lang="ts">
const traitStore = useTraitStore()
const { traits, loading, pagination } = storeToRefs(traitStore)

// SSR을 위한 데이터 로딩
const { data: initialTraits } = await useFetch('/api/traits', {
  baseURL: useRuntimeConfig().public.apiBase,
  query: {
    page_no: 1,
    page_size: 20,
  },
})

// 초기 데이터를 Store에 설정
if (initialTraits.value) {
  traits.value = initialTraits.value.items
  pagination.value.total = initialTraits.value.total_cnt
}

// 검색 파라미터
const searchParams = ref({
  search_keyword: '',
  trait_lcls: '',
  trait_mcls: '',
  aply_trgt: '',
  page_no: 1,
  page_size: 20,
})

// 검색 실행
async function handleSearch() {
  await traitStore.fetchTraits(searchParams.value)
}

// 페이지 변경
async function handlePageChange(pageNo: number) {
  searchParams.value.page_no = pageNo
  await handleSearch()
}
</script>

<template>
  <div>
    <h1>전역 특성 목록</h1>
    
    <!-- 검색 폼 -->
    <form @submit.prevent="handleSearch">
      <input v-model="searchParams.search_keyword" placeholder="검색어" />
      <button type="submit">검색</button>
    </form>

    <!-- 로딩 상태 -->
    <div v-if="loading">로딩 중...</div>

    <!-- 특성 목록 -->
    <ul v-else>
      <li v-for="trait in traits" :key="trait.trait_no">
        <NuxtLink :to="`/traits/${trait.trait_no}`">
          {{ trait.trait_nm }}
        </NuxtLink>
      </li>
    </ul>

    <!-- 페이지네이션 -->
    <Pagination 
      :total="pagination.total" 
      :page-size="pagination.pageSize"
      :current="pagination.pageNo"
      @change="handlePageChange"
    />
  </div>
</template>
```

## 참고사항

### TypeScript 타입 정의 전략
- API 응답 타입은 백엔드 VO 스키마와 일치
- Zod 스키마로부터 타입 추론 (`z.infer<typeof Schema>`)
- 폼 데이터 타입과 API 데이터 타입 분리

### 상태 관리 전략
- 페이지별 독립적인 Store 사용
- 공통 상태는 common Store에서 관리
- Store 간 의존성 최소화

### 성능 최적화
- useFetch의 캐싱 활용 (SSR)
- Pinia Store의 computed 활용
- 불필요한 리렌더링 방지 (toRefs, computed)
- 페이지네이션으로 대량 데이터 처리

### 보안 고려사항
- CORS 설정 (FastAPI)
- JWT 인증 구현 (추후)
- XSS 방지 (Vue의 자동 이스케이핑)
- CSRF 토큰 (필요시)
